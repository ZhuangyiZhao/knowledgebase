# 深圳后端求职 3 个月冲刺计划（字节后端版）

更新时间：2026-04-18  
适用对象：9 年经验后端工程师，目标城市深圳，目标公司优先字节跳动及同梯队大厂  
当前技术栈：MySQL / Kafka / Hive / HBase / Spark / Golang / Python / Redis

## 目标定义

目标不是单纯“背八股”，而是 3 个月后做到下面 4 件事同时在线：

1. 基础题答得稳，原理讲得清。
2. 项目题能深挖到架构、稳定性、性能、故障处理。
3. 系统设计题有套路，能讲权衡，不是堆名词。
4. 能跟上近期热点，尤其是字节偏爱的 AI 平台化、工程化、高并发、可观测性方向。

## 时间约束与使用方式

你每天只有 1 小时，因此必须用固定节奏，不临时发挥。

每日固定拆分：

- `20 分钟` 八股主线复习
- `20 分钟` 设计题 / 手写题 / 口述题
- `20 分钟` 项目复盘或近期热点补充

每周只学 6 天，第 7 天只做复盘，不新增内容。

## 三个月总路线

### 第 1 个月：补基础，建立标准答案

目标：把 OS / 网络 / MySQL / Redis / 基础算法打成一套稳定回答模板。

### 第 2 个月：中间件、分布式、工程化

目标：把 Kafka / 分布式一致性 / Go & Python 工程实践 / 服务治理讲透。

### 第 3 个月：项目拷打、系统设计、热点冲刺

目标：把“你做过什么”讲成“你为什么值得要”，并适配字节风格面试。

## 字节风格面试重点

字节通常不会停留在“背定义”，而会追问到工程细节。要重点准备这 5 类问题：

1. 你做过的系统规模有多大，峰值多少，瓶颈在哪。
2. 你主导过什么关键设计，为什么这样选，不这样选会怎样。
3. 一次线上故障你怎么发现、止损、定位、复盘。
4. 高并发下如何保证一致性、幂等、稳定性。
5. 你有没有做过性能优化、成本优化、治理建设。

## 近期热点追踪

以下内容建议重点补，因为它们很容易被面试官当作“你最近关注什么技术”的切口。

### 1. Kafka 4.0

关键点：

- 2025-03-18 Apache Kafka 4.0 发布
- 默认使用 KRaft，不再依赖 ZooKeeper
- 新一代 consumer rebalance protocol（KIP-848）是重要面试点

建议补到的程度：

- KRaft 和 ZooKeeper 模式的差异
- 为什么 Kafka 要去 ZooKeeper
- 新 rebalance 协议解决了什么问题
- 升级到 Kafka 4.0 要注意哪些兼容性问题

### 2. Redis 8

关键点：

- 2025-05-01 Redis 8 GA
- 官方强调性能提升、更多数据结构、向量检索相关能力

建议补到的程度：

- Redis 8 相比旧版本的价值点
- Redis 作为缓存之外还能做什么
- JSON / 向量 / 时序能力适合什么业务
- Redis 在 AI 应用链路中的角色

### 3. Spark 4.0

关键点：

- Spark 4.0 已发布
- VARIANT、Spark Connect、Streaming 能力增强值得了解

建议补到的程度：

- Spark Connect 是什么
- Structured Streaming 的状态管理解决什么问题
- Spark 4.0 对数据平台/数仓工程师的价值

### 4. Go 1.24

关键点：

- 2025-02-11 Go 1.24 发布
- 泛型 type alias、runtime 优化、tool 依赖管理是常见切入点

建议补到的程度：

- 你最近关注过哪些 Go 新变化
- Swiss Tables map 带来的意义
- runtime 2% 到 3% CPU 开销下降说明了什么

### 5. Kubernetes 1.33

关键点：

- 2025-04-23 Kubernetes 1.33 发布
- Sidecar Containers 稳定化、nftables kube-proxy、拓扑感知路由值得知道

建议补到的程度：

- 如果面平台岗，至少能说清 Pod 调度、探针、滚动发布、HPA
- 知道 kube-proxy、Service、Ingress 的基本流量路径

### 6. AI 平台化 / 大模型应用工程化

这是当前最值得额外加分的一块。

至少补到下面这些概念：

- RAG 的基本链路：切片、Embedding、召回、重排、生成
- 向量检索与传统倒排检索的区别
- Agent 的后端支撑：工具调用、状态管理、限流、审计、可观测性
- MCP 是什么，它在工具接入里的作用
- 大模型应用为什么特别需要缓存、限流、超时和成本控制
