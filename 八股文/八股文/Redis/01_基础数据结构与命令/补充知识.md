# Redis 常见数据类型速记

## 1. 一句话记忆

**Redis 基础类型 = 值、对象、队列、集合、排行榜。**  
**Redis 特殊类型 = BitMap、HyperLogLog、Geo、Stream。**

对应关系如下：

| 记忆词 | Redis 类型 | 核心特点 | 典型场景 |
|---|---|---|---|
| 一个值 | `String` | 单个 key 对应单个 value | 验证码、计数器、缓存 JSON |
| 一个对象 | `Hash` | 一个 key 下有多个字段 | 用户信息、商品字段 |
| 一串有序值 | `List` | 有序、可重复、两端操作 | 消息队列、任务队列 |
| 一堆不重复值 | `Set` | 无序、去重、支持集合运算 | 标签、好友、点赞用户 |
| 带分数的集合 | `ZSet` | 去重，按 score 排序 | 排行榜、热度榜、延迟队列 |
| 二值状态 | `Bitmap` | 用 bit 记录 0/1 | 签到、活跃状态 |
| 去重计数 | `HyperLogLog` | 低内存估算基数 | UV 统计 |
| 经纬度 | `Geo` | 存储地理位置并计算距离 | 附近的人、附近门店 |
| 消息流 | `Stream` | 追加写入、消费组 | 订单事件、日志流 |

---

## 2. 内在规律

Redis 的数据类型可以按“数据怎么组织”来理解：

```text
String：一个值
Hash：一个对象
List：有序可重复
Set：无序不重复
ZSet：不重复 + 可排序
```

特殊类型多数是围绕特定场景做的优化：

```text
Bitmap：用 bit 记录真假
HyperLogLog：用很少内存估算去重数量
Geo：存地理位置
Stream：存消息流
```

## 3. 类型示例

### 3.1 String：缓存验证码

```redis
SET verify:phone:13800138000 123456 EX 300
```

表示给手机号缓存 5 分钟验证码。

### 3.2 Hash：存用户信息

```redis
HSET user:1001 name "Tom" age 18 city "Shanghai"
```

适合存一个对象的多个字段。

### 3.3 List：消息队列

```redis
LPUSH queue:order order_001
RPOP queue:order
```

生产者从左侧写入，消费者从右侧取出，可以实现简单的先进先出队列。

### 3.4 Set：共同好友

```redis
SINTER user:1:friends user:2:friends
```

求用户 1 和用户 2 的共同好友。

### 3.5 ZSet：排行榜

```redis
ZADD rank:score 98 user1 95 user2 88 user3
ZREVRANGE rank:score 0 9 WITHSCORES
```

按分数倒序查询排行榜前 10 名。

### 3.6 Bitmap：用户签到

```redis
SETBIT sign:1001:2026-07 1 1
```

表示用户 1001 在 7 月第 2 天签到，因为 offset 从 0 开始。

### 3.7 HyperLogLog：统计网站 UV

```redis
PFADD uv:2026-07-02 user1 user2 user3
PFCOUNT uv:2026-07-02
```

用于估算当天独立访客数。

### 3.8 Geo：附近门店

```redis
GEOADD shop:location 121.4737 31.2304 shop1
GEOSEARCH shop:location FROMLONLAT 121.4737 31.2304 BYRADIUS 3 km
```

存储门店经纬度，并查询附近 3 公里内的门店。

### 3.9 Stream：订单消息流

```redis
XADD order:stream * orderId 1001 status created
```

可以配合消费组处理订单创建事件。

## 4. 面试回答模板

Redis 常见数据类型可以从数据结构角度理解：

> `String` 是单值，`Hash` 是对象，`List` 是有序列表，`Set` 是去重集合，`ZSet` 是带 score 的有序集合。除此之外，Redis 还提供 `Bitmap` 做二值状态统计，`HyperLogLog` 做基数统计，`Geo` 做地理位置，`Stream` 做消息流。

选类型时主要看访问模式：是否需要字段级更新、是否需要去重、是否需要排序、是否需要范围查询，以及是否是统计类场景。

## 5. 数据类型和底层编码

Redis 对外暴露的是数据类型，例如 `String`、`Hash`、`List`、`Set`、`ZSet`。  
压缩列表 `ziplist` 不是一种对外数据类型，而是 Redis 早期为了节省内存使用的底层紧凑编码结构。

可以这样区分：

```text
对外数据类型：String / Hash / List / Set / ZSet / Stream
底层编码结构：SDS / dict / intset / ziplist / listpack / quicklist / skiplist
```

早期 Redis 中，数据量较小、元素较短时，下面这些类型可能使用 `ziplist`：

| 对外类型 | 小数据量时的旧编码 | 数据量变大后的常见结构 |
|---|---|---|
| `Hash` | `ziplist` | `dict` |
| `List` | `ziplist` | `linkedlist`，后来演进为 `quicklist` |
| `ZSet` | `ziplist` | `skiplist` + `dict` |

新版 Redis 中，`ziplist` 已逐渐被 `listpack` 替代。面试回答时可以说：

> 压缩列表是 Redis 早期用于节省内存的底层编码，不是独立的数据类型。它曾用于小 Hash、小 List、小 ZSet；Redis 新版本更多使用 listpack，List 底层主要是 quicklist。
