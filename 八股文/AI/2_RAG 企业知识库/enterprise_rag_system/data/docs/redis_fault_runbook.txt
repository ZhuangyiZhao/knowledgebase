Redis 故障手册

缓存击穿

缓存击穿是指某个热点 key 在过期瞬间，大量并发请求同时没有命中缓存，直接打到数据库，导致数据库压力突然升高。处理思路是先止血再恢复缓存：入口可以做限流和降级，回源重建缓存时使用互斥锁、singleflight 或分布式锁，确保同一时间只有少量请求访问数据库。

对于特别热点的数据，可以使用逻辑过期。逻辑过期不是删除缓存，而是在 value 中保存过期时间，请求发现逻辑过期后仍然返回旧值，同时异步刷新缓存。这样可以避免热点 key 失效瞬间把数据库打满。

缓存雪崩

缓存雪崩通常是大量 key 在同一时间过期，或者 Redis 集群整体不可用，导致请求直接落到数据库。预防方式包括给过期时间增加随机抖动、热点数据预热、多级缓存、限流降级和 Redis 高可用部署。

大 key 排查

Redis 大 key 会导致网络传输变慢、删除阻塞、主从复制延迟和内存倾斜。可以使用 redis-cli --bigkeys、MEMORY USAGE、慢日志和业务 key 采样定位。处理方式包括拆分数据结构、异步删除、设置合理过期时间和限制 value 大小。
