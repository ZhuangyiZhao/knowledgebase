# 字节后端 200 题

更新时间：2026-04-18  
适用目标：深圳后端岗位，优先字节 / TikTok / 业务中台 / 基础架构 / AI Platform 方向

## 使用方式

这份题单不是让你一口气背完，而是建议这样使用：

1. 每天挑 `10 题`，先口述，再补标准答案。
2. 每周至少完成 `1 次` 30 分钟模拟，随机抽题。
3. 每道题回答时统一按这 4 步：
   - 先说定义或结论
   - 再说原理
   - 再说工程实践
   - 最后说边界和坑

## 最后提醒

如果面试目标是字节，这 200 题里不要只背“定义”。每题最好都补上你自己的项目例子，尤其是下面这些：

- MySQL 慢 SQL 优化案例
- Redis 热 key / 一致性治理案例
- Kafka 积压或重复消费案例
- Go 服务性能排障案例
- 一次线上故障止损和复盘案例
- 一个高并发系统设计与演进案例

真正拉开差距的，不是你会不会背，而是你能不能把“原理”讲成“我在线上就是这么做的”。
