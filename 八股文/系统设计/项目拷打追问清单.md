# 项目拷打追问清单

更新时间：2026-04-18  
适用目标：后端 / 平台 / 基础架构 / 数据平台 / AI Platform 面试

## 这份文档怎么用

项目面试不是讲故事，而是要让面试官相信这件事真的是你做的，而且你真的理解系统的关键权衡。

建议用法：

1. 先挑 `3 个项目`，每个项目单独套一遍这份清单。
2. 每个问题都至少准备 `30 秒版` 和 `2 分钟版` 两种回答。
3. 每次模拟面试，从不同维度随机抽 `15 个问题` 自问自答。

## 项目回答黄金结构

每个项目先用下面这个结构讲 2 分钟版本：

1. 背景：这是个什么业务，解决什么问题。
2. 规模：QPS、峰值、数据量、核心 SLA。
3. 职责：你负责哪一段，主导了什么。
4. 难点：高并发、一致性、实时性、稳定性、成本。
5. 方案：核心架构、关键选型、为什么这么选。
6. 结果：延迟、吞吐、成功率、成本有没有量化提升。
7. 复盘：最大的坑、最大的成长、如果重来会怎么做。

## 模拟拷打方法

建议每个项目至少做 3 轮模拟：

### 第 1 轮：2 分钟电梯稿

目标：讲清背景、规模、职责、难点、结果。

### 第 2 轮：15 题随机追问

从上面的 220 题里随机抽，回答时不看文档。

### 第 3 轮：故障与 trade-off 专项

专门练这几类：

- 为什么这么设计
- 哪里会出问题
- 线上怎么排障
- 为什么不用别的方案
- 如果流量翻倍怎么办

## 项目回答中的高频失分点

下面这些很容易被面试官判定为“做得不深”：

- 只会讲业务背景，不会讲规模
- 只会讲架构图，不会讲 trade-off
- 只会讲 happy path，不会讲故障
- 只会讲用了什么，不会讲为什么用
- 只会讲结果变好了，不会讲量化数据
- 只会说“我们”，说不清你具体负责了什么
- 说不清上线风险、压测结果、监控指标、回滚方案

## 你现在就该补齐的 3 个项目

建议优先准备：

1. 一个高并发核心链路项目
2. 一个稳定性治理或故障处置项目
3. 一个性能优化或架构演进项目

如果你后面要冲字节的 AI Platform / Dev Platform / 基础架构方向，再加一个：

4. 一个平台化、工具化、治理建设类项目

## 最终目标

练到下面这个状态，项目面试基本就稳很多了：

- 任何一个项目都能先讲 2 分钟版本
- 面试官从任何维度追问，你都能接住
- 每个项目至少有一个故障案例、一个优化案例、一个 trade-off 案例
- 每个项目都能量化结果
- 每个项目都能体现你的 owner、判断力和工程深度
